TGA Export Options

A Krita plugin that gives you full control over TGA export settings. It hooks into Krita's imageSaved signal and automatically shows an options dialog every time you save or export a .tga file.

How It Works

  1. Save or export a file as .tga using File → Save As or File → Export.
  2. Krita writes the TGA file to disk.
  3. The plugin detects the save and opens a dialog with export options.
  4. Choose your settings and click OK, or click Skip to leave the file unchanged.
  5. The plugin modifies the TGA file on disk according to your choices.
Tip: The plugin does nothing to non-TGA files. It only activates for .tga saves.

Options

Option Values Description
Origin Bottom-Left / Top-Left Sets the pixel origin point.
Bottom-Left is the TGA standard, used by most image editors.
Top-Left is used by OpenGL, DirectX, and many game engines.
Bit Depth Keep as is / 32 bit / 24 bit Keep as is — don't change the bit depth.
32 bit (RGBA) — adds an alpha channel (0xFF) if missing.
24 bit (RGB) — strips the alpha channel for a smaller file.
Note: 16-bit TGAs cannot be converted and are kept as is.
RLE Compression On / Off Run-Length Encoding compression. Can significantly reduce file size for images with large areas of the same color.
TGA 2.0 Footer On / Off Adds the TRUEVISION-XFILE signature at the end of the file. Many modern applications check for this to identify TGA 2.0 files. Recommended to keep enabled.
Comment (Image ID) Up to 255 characters A text string embedded in the TGA header. Only ASCII characters are supported; non-ASCII characters are replaced with ?.

Persistent Settings

All options (origin, bit depth, RLE, footer, comment, and last used directory) are saved between Krita sessions using QSettings. When the dialog opens next time, everything will be exactly as you left it.

Supported TGA Formats

  • Uncompressed: Color-mapped (type 1), True-color (type 2), Grayscale (type 3)
  • RLE-compressed: Color-mapped (type 9), True-color (type 10), Grayscale (type 11)
  • Pixel depths: 8, 16, 24, 32 bits per pixel
  • All origin positions: Bottom-Left, Bottom-Right, Top-Left, Top-Right

Troubleshooting

If the dialog does not appear when saving a TGA file:

  1. Make sure the plugin is enabled in Settings → Configure Krita → Python Plugin Manager.
  2. Restart Krita twice after enabling the plugin.
  3. Launch Krita from the command line to see debug messages:
    • Windows: "C:\Program Files\Krita (x64)\bin\krita.exe"
    • Linux/macOS: krita
  4. Look for [TGA Export Options] messages in the console output.
Common mistake: The plugin folder must be named tga_export_options (all lowercase). Using Tga_Export_Options or any other case will cause "No module named" errors.
